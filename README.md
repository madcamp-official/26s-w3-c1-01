## 핸즈프리 스마트폰 컨트롤러 — 테스터 안내

안녕하세요. 저희는 KAIST 몰입캠프에 참여하고 있는 학생 팀입니다. 짧은 기간 동안 팀 단위로 서비스를 기획하고 직접 구현해보는 프로그램입니다. 이번 프로젝트에서 저희 팀은 1주일이라는 제한된 시간 안에 손을 자유롭게 쓰기 어려운 상황에서 스마트폰을 더 편안하게 사용할 수 있는 방법을 실험하고 있습니다. 저희가 만든 앱은 상업적 목적의 서비스나 감시 도구가 아니라, **어떤 컨텐츠가 실제 사용자에게 편안함을 주고 짧은 개발 기간 안에서도 많은 사람에게 공개 배포 가능한 수준이 될지** 를 확인하기 위한 실험적 프로젝트입니다.

**목적**

손 쓰기 힘든 상황(요리·운동 중 등)에서 얼굴 방향으로 포인터를 움직이고, 음성 명령으로 터치·스크롤·드래그하는 실험적 컨트롤러입니다.

### 사용 순서

1. 앱 실행 → 접근성 서비스 켜기 → 오버레이 권한 허용 → 카메라/마이크 권한 허용
2. 화면의 9개 지점을 순서대로 바라보며 캘리브레이션
3. 완료되면 다른 앱으로 넘어가서 바로 사용 가능

### 주요 음성 명령

- **터치 / 클릭**
- **아래로·내려, 위로·올려** (스크롤)
- **잡아**(드래그 시작), **놓아·놔**(종료)
- **멈춰**, **다시 시작**,
- **잠금**, **해제** (기능 일시 정지, 해제)

### 권한이 필요한 이유

**카메라 권한** : 얼굴 방향을 추적해 포인터 위치를 계산하기 위해 필요합니다.

**마이크 권한** : “터치”, “내려”, “올려” 같은 음성 명령을 인식하기 위해 필요합니다.

**오버레이 권한** : 다른 앱 위에 포인터와 상태 표시를 띄우기 위해 필요합니다.

**접근성 서비스 권한** : 실제 터치, 스크롤, 드래그 동작을 실행하기 위해 필요합니다.

접근성 서비스를 켜지 않으면 음성 인식은 되어도 실제 터치나 스크롤은 실행되지 않습니다.

### 개인정보 및 보안 안내

이 앱은 악성 코드나 감시 앱이 아닙니다. 테스트 목적과 수집 항목을 투명하게 공개합니다.

- 카메라 영상은 저장하지 않습니다.
- 얼굴 이미지는 저장하지 않습니다.
- 음성 녹음 파일은 저장하지 않습니다.
- 화면 내용은 수집하지 않습니다.
- 이름, 전화번호, 연락처, 메시지 내용 같은 개인정보는 수집하지 않습니다.
- 진단 데이터 전송은 선택 사항입니다.
- 진단 데이터는 사용자가 동의 체크를 했을 때만 전송됩니다.

수집될 수 있는 진단 데이터는 다음과 같습니다.

- 앱 실행 여부
- 캘리브레이션 시작/완료/실패 여부
- 명령 실행 성공/실패 여부
- 명령 종류별 실행 횟수
- 포인터 처리 FPS
- 얼굴 미검출 빈도
- 음성 인식 실패율
- 앱 오류/크래시 정보
- 사용자가 직접 입력한 피드백

이 데이터는 앱이 실제 환경에서 잘 작동하는지 확인하고, 공개 배포 가능성을 판단하기 위한 용도로만 사용됩니다.

### 참고

현재 앱은 실험적 테스트 버전입니다. 기기 성능, Android 버전, 주변 소음, 조명, 얼굴 위치에 따라 인식 성능이 달라질 수 있습니다.

사용 중 불편했던 점이나 명령이 잘 인식되지 않았던 상황을 피드백으로 남겨 주시면 개선에 큰 도움이 됩니다.
