# 핸즈프리 컨트롤러

**얼굴로 포인터를 옮기고, 목소리로 터치하는 안드로이드 앱.**

전면 카메라로 고개 방향을 추정해 화면 위 가상 포인터를 움직이고, 음성 명령으로
실제 터치 이벤트를 발생시킵니다. 요리 중이거나, 태블릿을 거치해 두고 보거나,
손을 쓰기 어려운 상황에서 화면을 만지지 않고 조작하기 위해 만들었습니다.

> 몰입캠프 3주차 공통과제 III · 4인 · 5일

## 📥 설치

**[최신 버전 내려받기 (Releases)](../../releases/latest)** → `app-release.apk`

설치와 권한 설정에 몇 단계가 필요합니다.
**[설치 및 사용 가이드](docs/INSTALL.md)를 먼저 읽어 주세요.**

| | |
| --- | --- |
| 안드로이드 | 8.0 이상 · **전면 카메라 필수** |
| 네트워크 | 음성 인식에 필요 |
| 용량 | APK 약 55MB |

**아이폰에서는 동작하지 않습니다.** iOS는 다른 앱 위에 터치를 발생시키는 것을
허용하지 않습니다.

## 핵심 설계: 시선만으로는 아무것도 실행되지 않는다

화면을 그냥 보는 것과 조작하려는 의도를 구분해야 합니다. 그래서 **실행 트리거는
전부 음성**이고, 일정 시간 응시하면 자동 실행되는 방식(dwell)은 의도적으로
넣지 않았습니다. 넣는 순간 "보기만 해도 눌리는" 앱이 됩니다.

포인터 이동(얼굴)과 실행(음성)은 병렬로 돌고, **명령이 인식된 시점의 포인터 좌표**를
기준으로 동작합니다.

## 음성 명령

| 하고 싶은 것 | 말할 것 |
| --- | --- |
| 누르기 | "터치" / "클릭" |
| 뒤로 가기 | "취소" |
| 스크롤 | "위로" / "아래로" (+ "조금", "크게") |
| 좌우 이동 | "다음" / "이전" |
| 끌기 | "잡아" → "놓아" |
| 일시 정지 | "멈춰" → "다시 시작" |
| 잠금 | "잠금" → "해제" |

음성 인식이 반복 실패해도 앱에 갇히지 않도록 **화면 내 수동 잠금 해제 버튼이
항상 표시**됩니다.

## 개인정보

- **카메라 영상·얼굴 이미지·음성 녹음을 저장하거나 전송하지 않습니다.**
  좌표를 계산하는 즉시 버립니다.
- **화면 내용을 읽지 않습니다.** 접근성 이벤트 구독을 꺼 두었습니다
  ([`accessibility_service_config.xml`](app/src/main/res/xml/accessibility_service_config.xml)).
- 진단 데이터 전송은 **기본이 꺼져 있고**, 켜신 경우에만 익명 통계를 보냅니다.
- 기기에 남는 것은 캘리브레이션 수치와 무작위 식별자뿐이며, 앱을 삭제하면 사라집니다.

접근성 서비스 권한을 요구합니다 — 다른 앱 위에 터치를 발생시키려면 안드로이드에서
다른 방법이 없습니다. **낯선 앱에 이 권한을 주는 것을 망설이는 것이 정상이고,
그래서 소스 전체를 공개합니다.**

## 기술 구성

**Kotlin · CameraX · MediaPipe Face Landmarker · AccessibilityService**

```
카메라 → 얼굴 랜드마크 → yaw/pitch 추정 → 정규화·매핑 → smoothing → 포인터 좌표
                                                                         ↓
음성 → 명령어 사전 매칭 → 상태 검증(State Manager) → 터치 이벤트 주입 → OS
```

- **MediaPipe를 쓴 이유는 홍채입니다.** ML Kit은 더 가볍고 머리 각도를 바로 주지만
  홍채·시선이 없어서 시선 보조 가중치를 구현할 수 없습니다.
- 전부 **온디바이스**로 동작합니다. 단, 음성 인식만 예외입니다 (아래 참고).
- 포인터는 앱을 벗어나도 유지됩니다 — 카메라·음성·오케스트레이터의 수명을
  포그라운드 서비스에 묶어 두었기 때문입니다.

### 빌드

```bash
./gradlew assembleDebug
./gradlew testDebugUnitTest
```

`app/src/main/assets/face_landmarker.task`(3.7MB) 모델이 저장소에 포함되어 있어
별도 준비 없이 빌드됩니다. AGP 9.3.0 / Gradle 9.5.1 / compileSdk 36.

> **AGP 9는 Kotlin이 내장이라 `org.jetbrains.kotlin.android` 플러그인을 추가하면
> Sync가 실패합니다.**

## 알려진 한계

솔직하게 적습니다. 발견하고 실망하는 것보다 미리 아는 게 낫다고 생각합니다.

- **눈만으로는 조작할 수 없습니다.** 고개 방향이 주 입력이고 눈동자는 미세 조정
  역할입니다. 일반 전면 카메라로 시선을 정밀 추적하는 것은 물리적으로 어렵습니다.
- **음성 인식에 인터넷이 필요합니다.** 온디바이스로 만들려 했지만 한국어 오프라인
  모델이 없는 기기에서 인식이 전혀 되지 않아 온라인 방식으로 되돌렸습니다.
- **삼성 런처 홈 화면에서 좌우 페이지 넘김이 동작하지 않습니다.** 일반 앱(갤러리 등)에서는
  됩니다. 런처가 자체 제스처 처리를 써서 주입된 신호를 받지 않는 것으로 파악했습니다.
- **주변 사람의 발화에 반응할 수 있습니다.** 화자 인식은 넣지 않았습니다.
- 카메라와 마이크를 계속 켜 두기 때문에 **발열이 있습니다.**
