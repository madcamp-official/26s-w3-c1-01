# 기능 명세서 (Functional Specification)

## 핸즈프리 스마트폰 컨트롤러 (Hands-free Smartphone Controller)

- 문서 버전: v1.0
- 작성 기준: 기획안(핸즈프리 스마트폰 컨트롤러) v1.0
- 대상 범위: MVP(최초 실험 버전)

---

## 1. 문서 개요

### 1.1 목적

본 문서는 기획안에 정의된 "핸즈프리 스마트폰 컨트롤러"의 기획 내용을 개발 가능한 수준의 기능 단위로 구체화하여, 각 기능의 입력·처리·출력·예외 조건·상태 전이를 명확히 정의하는 것을 목적으로 한다.

### 1.2 범위

본 문서는 MVP 범위에 한정한다.

**포함 범위**

- 얼굴/시선 기반 포인터 이동
- 음성 명령 기반 터치, 드래그, 스크롤
- 일시정지/재개, 입력 잠금
- 초기 캘리브레이션
- 위 기능에 필요한 음성 인식 처리, 상태 관리, 예외 처리

**제외 범위 (Phase 2 이후)**

- 특정 앱(유튜브, 웹툰, 도서 앱 등) 전용 최적화 기능
- 정밀 아이트래킹(안구 각막 반사 기반 등 고정밀 트래킹)
- 다중 사용자 프로필 동기화(클라우드 저장)
- 제스처(표정, 눈 깜빡임 등) 기반 명령 확장

### 1.3 용어 정의

| 용어 | 정의 |
| --- | --- |
| 포인터(Pointer) | 사용자의 얼굴/시선 방향 추정 결과를 화면 좌표에 매핑하여 표시하는 가상 커서 |
| 트리거 명령 | 포인터 위치에서 실제 동작(터치, 드래그 등)을 실행시키는 음성 명령 |
| 캘리브레이션 | 사용자의 얼굴/눈 움직임 범위와 화면 좌표를 매핑하는 초기 보정 과정 |
| 활성 상태(ACTIVE) | 포인터 이동 및 음성 명령이 모두 처리되는 상태 |
| 일시정지 상태(PAUSED) | 포인터 이동은 멈추고, 재개 명령만 인식하는 상태 |
| 잠금 상태(LOCKED) | 모든 입력(포인터, 음성)이 무시되는 상태 |
| Dwell(체류) | 포인터가 특정 영역에 일정 시간 이상 머무는 것(트리거 보조 용도로만 사용, 자동 실행에는 미사용) |

### 1.4 참고 문서

- 기획안(핸즈프리 스마트폰 컨트롤러) v1.0

## 2. 시스템 개요

### 2.1 주요 모듈 구성

| 모듈 | 역할 |
| --- | --- |
| 얼굴/시선 추정 모듈 (Face Tracker) | 전면 카메라 입력으로부터 얼굴 방향, 눈 위치, 머리 움직임 추정 |
| 포인터 매핑 모듈 (Pointer Mapper) | 추정된 방향 값을 화면 좌표로 변환, smoothing 및 속도 조절 적용 |
| 음성 인식 모듈 (Voice Recognizer) | 사용자 발화를 텍스트/명령으로 변환 |
| 명령 해석 모듈 (Command Interpreter) | 인식된 텍스트를 사전 정의된 명령으로 매핑, 동의어 처리 |
| 상태 관리 모듈 (State Manager) | ACTIVE/PAUSED/LOCKED/CALIBRATING/DRAGGING 등 전체 상태 전이 관리 |
| 입력 실행 모듈 (Input Injector) | OS 접근성 서비스(또는 이에 준하는 API)를 통해 실제 탭/드래그/스크롤 이벤트 발생 |
| 캘리브레이션 모듈 (Calibration Manager) | 초기 보정 플로우 진행 및 사용자 프로파일 저장 |
| UI 오버레이 모듈 (Overlay UI) | 포인터, 상태 인디케이터, 설정 화면 등 화면 표시 요소 관리 |

### 2.2 전체 동작 흐름

```
[카메라 입력] → [얼굴/시선 추정] → [포인터 좌표 계산] → [오버레이에 포인터 표시]
                                                              ↑
[마이크 입력] → [음성 인식] → [명령 해석] → [상태 확인] → [입력 실행] → [OS 터치/드래그/스크롤 이벤트]
```

포인터 이동과 음성 명령 처리는 병렬로 동작하며, 명령 실행 시점의 포인터 좌표를 기준으로 동작이 수행된다.

### 2.3 전체 상태 정의 (State Machine)

| 상태 | 설명 | 진입 조건 | 진출 조건 |
| --- | --- | --- | --- |
| CALIBRATING | 초기 보정 진행 중 | 앱 최초 실행, 재보정 요청 | 보정 완료 |
| ACTIVE | 포인터 이동 + 음성 명령 모두 처리 | 보정 완료, "다시 시작", "해제" | "멈춰", "잠금", 드래그 시작 시 DRAGGING 진입 |
| PAUSED | 포인터 고정, "다시 시작" 명령만 인식 | "멈춰" | "다시 시작" |
| LOCKED | 모든 입력 무시, "해제" 명령만 인식 | "잠금" | "해제" |
| DRAGGING | 드래그 진행 중, 포인터 이동은 되지만 트리거 명령은 드래그 관련 명령만 처리 | "잡아"/"드래그 시작" | "놓아"(완료), "드래그 취소"(취소) |

상태 전이 다이어그램(요약):

```
CALIBRATING --(보정완료)--> ACTIVE
ACTIVE --(멈춰)--> PAUSED --(다시 시작)--> ACTIVE
ACTIVE --(잠금)--> LOCKED --(해제)--> ACTIVE
ACTIVE --(잡아/드래그 시작)--> DRAGGING --(놓아)--> ACTIVE
DRAGGING --(드래그 취소)--> ACTIVE
```

## 3. 기능 명세

각 기능은 FR(Functional Requirement) 번호로 관리한다.

### FR-001 포인터 이동

**개요**
전면 카메라를 통해 얼굴 방향과 눈 위치를 추정하여 화면 위 포인터를 이동시킨다.

**입력**

- 전면 카메라 프레임 스트림 (권장 15fps 이상)
- 캘리브레이션 프로파일 (사용자별 저장값)

**처리 로직**

1. 프레임에서 얼굴 랜드마크 검출
2. 얼굴 방향(yaw/pitch) 및 눈 위치 추정
3. 캘리브레이션 프로파일 기준으로 정규화
4. 정규화된 값을 화면 좌표(x, y)로 매핑
5. Smoothing 필터(예: 이동 평균 또는 저역 통과 필터) 적용하여 떨림 감소
6. 포인터 이동 속도 설정값에 따라 좌표 변화량 스케일링

**출력**

- 화면 좌표 (x, y) — Overlay UI에 실시간 반영

**세부 파라미터**

| 파라미터 | 설명 | 기본값(안) | 조정 범위 |
| --- | --- | --- | --- |
| 이동 속도(Sensitivity) | 얼굴 각도 변화 대비 포인터 이동량 | 중간 | 낮음/중간/높음 (3단계) |
| Smoothing 강도 | 떨림 저감 필터 강도 | 중간 | 낮음/중간/높음 (3단계) |
| 화면 구역 이동 | 화면을 N×N 구역으로 나누어 큰 폭으로 이동 | 3×3 | 2×2 ~ 4×4 |
| 시선 보조 가중치 | 얼굴 방향 대비 시선 방향 반영 비율 | 낮음 | 0~100% |

**예외 처리**

| 상황 | 처리 방안 |
| --- | --- |
| 얼굴 미검출(프레임 내 얼굴 없음) | 포인터를 마지막 위치에 고정, 화면에 "얼굴 인식 안됨" 안내 표시 |
| 저조도 환경 | 밝기 부족 안내 메시지 표시, 필요 시 화면 밝기 자동 보정 제안 |
| 얼굴 각도가 캘리브레이션 범위를 벗어남 | 포인터를 화면 경계값으로 클램핑(고정) |
| 카메라 권한 거부 | 기능 사용 불가 안내 및 권한 설정 화면으로 유도 |

### FR-002 기본 터치

**개요**
음성 명령으로 현재 포인터 위치에서 탭(터치) 이벤트를 실행한다.

**입력**

- 음성 명령: "터치", "클릭"
- 현재 포인터 좌표

**처리 로직**

1. 명령 해석 모듈이 발화를 "터치" 명령으로 인식
2. 현재 상태가 ACTIVE인지 확인 (PAUSED/LOCKED이면 무시)
3. 현재 포인터 좌표에서 OS 탭 이벤트 생성
4. 실행 결과에 대한 시각/청각 피드백 제공(예: 포인터 클릭 애니메이션)

**출력**

- 포인터 좌표 (x, y)에서의 탭 이벤트
- 실행 피드백(애니메이션 또는 짧은 진동/소리)

**지원 명령**

| 음성 명령 | 동작 | 비고 |
| --- | --- | --- |
| 터치 | 현재 위치 탭 | 기본 명령 |
| 클릭 | 현재 위치 탭 | "터치"의 동의어로 처리 |
| 취소 | 현재 동작 취소(뒤로가기) | OS 뒤로가기 이벤트와 매핑 |

**예외 처리**

- PAUSED/LOCKED 상태에서 명령 인식 시: 실행하지 않고 현재 상태 안내만 표시
- 포인터가 화면 경계 밖(클램핑된 상태)일 경우: 가장 가까운 유효 좌표에서 탭 실행

### FR-003 드래그

**개요**
포인터 이동과 음성 명령을 조합하여 드래그 동작(길게 눌러 이동)을 수행한다.

**입력**

- 음성 명령: "잡아"/"드래그 시작", "놓아", "드래그 취소"
- 드래그 진행 중 포인터 좌표 변화

**처리 로직**

1. "잡아" 또는 "드래그 시작" 인식 시 상태를 DRAGGING으로 전이하고 현재 좌표를 드래그 시작점으로 기록
2. DRAGGING 상태에서는 포인터 이동에 따라 실시간으로 드래그 경로(터치 다운 유지 + 좌표 갱신) 생성
3. "놓아" 인식 시 현재 좌표에서 터치 업 이벤트 발생, 상태를 ACTIVE로 복귀
4. "드래그 취소" 인식 시 시작점으로 포인터를 되돌리고 터치 업 없이 드래그 취소, 상태를 ACTIVE로 복귀

**출력**

- 드래그 시작/이동/종료에 대응하는 OS 터치 다운 → 이동 → 업 이벤트 시퀀스

**지원 명령**

| 음성 명령 | 동작 |
| --- | --- |
| 잡아 | 드래그 시작 |
| 드래그 시작 | 드래그 시작 |
| 놓아 | 드래그 종료 |
| 드래그 취소 | 드래그 취소 |

**상태 다이어그램**

```
ACTIVE --"잡아"/"드래그 시작"--> DRAGGING
DRAGGING --"놓아"--> ACTIVE (터치 업, 이동 확정)
DRAGGING --"드래그 취소"--> ACTIVE (시작점 복귀, 이동 취소)
```

**예외 처리**

| 상황 | 처리 방안 |
| --- | --- |
| DRAGGING 상태에서 "터치"/"클릭" 발화 | 무시 또는 "드래그 중입니다" 안내 |
| DRAGGING 중 얼굴 미검출 | 포인터를 마지막 유효 좌표에 고정, 드래그 상태 유지(자동 취소하지 않음) |
| DRAGGING 상태가 일정 시간(예: 30초) 이상 지속 | 자동으로 "드래그 취소" 처리 후 안내 메시지 표시 |

### FR-004 스크롤

**개요**
음성 명령으로 화면 스크롤(상/하) 동작을 수행한다.

**입력**

- 음성 명령: "아래로", "위로", "조금 아래로", "조금 위로", "크게 아래로", "크게 위로"

**처리 로직**

1. 명령 해석 모듈이 방향(위/아래)과 강도(기본/조금/크게)를 함께 파싱
2. 강도에 따라 스크롤 거리(픽셀 또는 화면 비율) 결정
3. 포인터 현재 좌표를 기준점으로 OS 스크롤(스와이프) 이벤트 생성

**출력**

- 지정된 방향/강도에 대응하는 스와이프 이벤트

**지원 명령 및 스크롤 강도(안)**

| 음성 명령 | 동작 | 스크롤 거리(화면 높이 대비) |
| --- | --- | --- |
| 아래로 | 아래로 스크롤(기본) | 약 50% |
| 위로 | 위로 스크롤(기본) | 약 50% |
| 조금 아래로 | 짧게 아래 스크롤 | 약 20% |
| 조금 위로 | 짧게 위 스크롤 | 약 20% |
| 크게 아래로 | 길게 아래 스크롤 | 약 80% |
| 크게 위로 | 길게 위 스크롤 | 약 80% |

**예외 처리**

- 스크롤 불가능한 화면(더 이상 스크롤할 콘텐츠 없음): 정상 스와이프 이벤트만 전달하고 별도 오류 처리는 하지 않음(OS/앱단 처리에 위임)
- 연속 발화("아래로 아래로")는 각각 개별 명령으로 순차 처리

### FR-005 일시정지 및 재개

**개요**
의도하지 않은 조작을 방지하기 위해 컨트롤러 전체를 일시정지하거나 입력을 잠글 수 있다.

**입력**

- 음성 명령: "멈춰", "다시 시작", "잠금", "해제"

**처리 로직**

1. "멈춰" 인식 시: 포인터 이동 갱신 중단(마지막 좌표에 고정 표시), 상태를 PAUSED로 전이. 이 상태에서는 "다시 시작"만 인식
2. "다시 시작" 인식 시: 포인터 이동 및 전체 명령 처리 재개, 상태를 ACTIVE로 전이
3. "잠금" 인식 시: 포인터 오버레이 자체를 숨기거나 반투명 처리, 모든 음성 명령을 무시하되 "해제" 명령만 예외적으로 인식
4. "해제" 인식 시: 잠금 해제, 상태를 ACTIVE로 복귀

**출력**

- 상태 변경에 따른 오버레이 UI 변화(포인터 표시/숨김, 상태 인디케이터 색상 변경)

**지원 명령**

| 음성 명령 | 동작 |
| --- | --- |
| 멈춰 | 포인터 이동 및 입력 일시 정지 |
| 다시 시작 | 컨트롤러 재개 |
| 잠금 | 입력 잠금 |
| 해제 | 입력 잠금 해제 |

**PAUSED vs LOCKED 차이**

| 구분 | PAUSED | LOCKED |
| --- | --- | --- |
| 포인터 표시 | 마지막 위치에 고정 표시 | 숨김 또는 반투명 처리 |
| 인식되는 명령 | "다시 시작"만 | "해제"만 |
| 용도 | 일시적으로 조작을 멈추고 화면만 볼 때 | 장시간 미사용, 오조작 완전 차단이 필요할 때 |

**예외 처리**

- LOCKED 상태에서 "다시 시작" 등 다른 명령 발화 시: 무시하고 "잠금 해제가 필요합니다" 안내
- 음성 인식 오류로 "해제"가 반복 실패할 경우를 대비해, 화면 내 수동 잠금 해제 버튼(물리 터치)을 항상 노출

### FR-006 캘리브레이션

**개요**
사용자별 얼굴 위치, 카메라 각도, 화면 크기, 사용 자세 차이를 보정하기 위한 초기 설정 과정.

**입력**

- 캘리브레이션 진행 중 사용자의 얼굴/시선 데이터 (9개 기준점 응시 시)

**처리 절차**

| 단계 | 내용 | 화면 안내 |
| --- | --- | --- |
| 1 | 기기 거치 및 얼굴 정렬 안내 | "기기를 거치하고 얼굴이 화면 중앙에 오도록 해주세요" |
| 2 | 화면 중앙 응시 | 중앙 기준점 표시 |
| 3 | 상/하/좌/우 및 대각선 총 9개 기준점 순차 응시 | 기준점을 순서대로 하이라이트, 응시 완료 시 다음 기준점으로 이동 |
| 4 | 얼굴/눈 움직임 범위 저장 | 진행률 표시 |
| 5 | 테스트 포인터로 민감도 조정 | 사용자가 직접 포인터를 움직여보고 이동 속도/스무딩 값을 조정 |

**출력**

- 사용자 캘리브레이션 프로파일 (기기 로컬 저장)

**캘리브레이션 데이터 스키마(안)**

| 필드명 | 설명 |
| --- | --- |
| profile_id | 프로파일 식별자 |
| reference_points[9] | 9개 기준점별 얼굴/눈 방향 원시값 |
| face_range_yaw_min/max | 얼굴 좌우 회전 범위 |
| face_range_pitch_min/max | 얼굴 상하 회전 범위 |
| sensitivity_level | 사용자가 선택한 이동 속도 단계 |
| smoothing_level | 사용자가 선택한 스무딩 단계 |
| created_at / updated_at | 생성/수정 일시 |

**예외 처리**

| 상황 | 처리 방안 |
| --- | --- |
| 특정 기준점에서 얼굴 미검출 지속(예: 5초 이상) | 해당 단계 재시도 안내, 3회 실패 시 처음부터 재시작 안내 |
| 캘리브레이션 중도 이탈 | 임시 저장하지 않고 폐기, 다음 실행 시 재시작 필요 |
| 캘리브레이션 미완료 상태에서 앱 실행 | 자동으로 캘리브레이션 플로우로 진입, ACTIVE 상태 진입 차단 |
| 조명/카메라 각도 문제로 반복 실패 | 조명 개선, 거치 각도 조정 등 가이드 문구 제공 |

**재보정**

- 설정 메뉴에서 "재보정" 메뉴 선택 시 기존 프로파일을 새 프로파일로 덮어씀
- 재보정 중에는 기존 프로파일로 계속 조작 가능(재보정 완료 후 교체)

### FR-007 음성 명령 인식 및 해석

**개요**
사용자의 발화를 인식하고 사전 정의된 명령 체계로 매핑한다. 모든 실제 조작(터치, 드래그, 스크롤 등)의 트리거는 음성으로만 처리하며, 시선/얼굴 위치만으로 자동 실행되지 않는다.

**처리 로직**

1. 마이크 입력을 실시간 음성 인식 엔진에 전달
2. 인식된 텍스트를 사전 정의된 명령어 사전과 매칭 (동의어 포함)
3. 매칭된 명령을 현재 상태(State Manager)에 전달하여 유효성 검증 후 실행

**명령어 사전 관리 원칙**

- 각 명령은 최소 1개 이상의 동의어를 지원한다 (예: "터치" = "클릭")
- 명령어 사전은 설정 화면에서 사용자가 커스텀 명령어를 추가/변경할 수 있도록 확장 가능한 구조로 설계한다 (MVP에서는 기본 사전만 제공, 커스터마이징은 Phase 2)

**예외 처리**

| 상황 | 처리 방안 |
| --- | --- |
| 인식된 텍스트가 사전에 없음 | 아무 동작도 실행하지 않고, 화면 하단에 "인식된 명령 없음" 짧게 표시 |
| 소음이 많은 환경 | 인식 신뢰도 임계값 이하일 경우 명령 무시, 재발화 유도 |
| 주변 사람의 발화로 인한 오작동 | (Phase 2) 화자 인식/음성 등록 기반 필터링 검토, MVP에서는 임계값 기반 필터링만 적용 |
| 마이크 권한 거부 | 기능 사용 불가 안내 및 권한 설정 화면으로 유도 |

### FR-008 전체 상태 관리 및 오버레이 표시

**개요**
현재 컨트롤러 상태(CALIBRATING/ACTIVE/PAUSED/LOCKED/DRAGGING)를 사용자가 항상 인지할 수 있도록 오버레이로 표시한다.

**상태별 UI 표시(안)**

| 상태 | 포인터 표시 | 상태 인디케이터 |
| --- | --- | --- |
| ACTIVE | 실시간 이동, 기본 색상 | 초록색 아이콘 |
| PAUSED | 마지막 위치 고정, 반투명 | 노란색 아이콘 + "일시정지" 텍스트 |
| LOCKED | 숨김 또는 매우 반투명 | 회색 아이콘 + "잠김" 텍스트 |
| DRAGGING | 이동 중, 강조 색상(예: 테두리 강조) | 파란색 아이콘 + "드래그 중" 텍스트 |
| CALIBRATING | 기준점 표시 전용 UI | 진행률 표시 |

**예외 처리**

- 상태 전이 중 충돌(예: 드래그 중 "멈춰" 발화) 시 우선순위: 안전을 위해 "멈춰"를 우선 처리하고 드래그는 자동 취소 처리 후 PAUSED로 전이
