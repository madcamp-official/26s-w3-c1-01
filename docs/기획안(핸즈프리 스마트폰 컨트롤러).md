# 기획안(핸즈프리 스마트폰 컨트롤러)

# 1. 주제

핸즈프리 스마트폰 컨트롤러는 사용자의 시선, 음성 입력을 활용해 스마트폰이나 태블릿을 손 없이 조작할 수 있게 돕는 모바일 앱이다.

앱은 기본적으로 화면 위에 가상 커서 또는 선택 포인터를 표시하고, 사용자의 얼굴 방향이나 시선 방향에 따라 포인터를 이동시킨다. 사용자는 음성 명령을 통해 터치, 드래그, 스크롤 같은 기본 조작을 수행할 수 있다. 

초기 버전은 모든 앱을 완벽하게 제어하는 범용 접근성 도구보다는, 기본 터치 조작을 손 없이 수행하는 실험적 컨트롤러를 목표로 한다. 이후에는 특정 앱이나 사용 상황에 맞춘 기능을 확장할 수 있다.(ex. 유튜브, 웹툰, 도서 앱 등)

# 2. 목적

이 프로젝트의 목적은 손을 사용하기 어렵거나, 손을 사용하지 않고도 스마트폰을 조작하고 싶은 사용자를 위한 새로운 입력 방식을 제안하는 것이다.

스마트폰은 대부분의 조작이 터치 중심으로 설계되어 있다. 그러나 다음과 같은 상황에서는 터치 조작이 불편하거나 불가능할 수 있다.

- 손이나 팔 사용이 제한적인 사용자
- 일시적으로 손을 사용할 수 없는 사용자
- 요리, 운동, 작업 중처럼 손이 젖었거나 바쁜 상황
- 태블릿을 거치해두고 멀리서 조작하고 싶은 상황
- 접근성 기술이나 새로운 인터페이스에 관심 있는 사용자

핸즈프리 스마트폰 컨트롤러는 이러한 상황에서 사용자가 화면을 바라보고 말하는 것만으로 기본적인 스마트폰 조작을 수행할 수 있도록 돕는다.

# 3. 핵심 컨셉

**> 얼굴과 시선으로 위치를 정하고, 목소리로 터치한다.**

사용자는 스마트폰 또는 태블릿 화면을 바라본다. 앱은 전면 카메라를 통해 사용자의 얼굴 방향, 눈 위치, 머리 움직임을 추정해 화면 위의 포인터를 이동시킨다. 사용자가 "터치", "누르기", "드래그 시작", "놓기" 같은 명령을 말하면 해당 위치에서 터치 동작이 실행된다.

핵심은 시선만으로 모든 조작을 자동 실행하지 않는 것이다. 단순히 화면을 보는 행동과 실제로 조작하려는 의도를 구분하기 위해, 실행 명령은 음성으로 처리한다.

# 4. 주요 기능

## 4.1. 포인터 이동

전면 카메라를 통해 사용자의 얼굴 방향과 눈 위치를 추정하고, 화면 위의 가상 포인터를 이동시킨다.

MVP에서는 정밀한 아이트래킹보다 안정적인 조작감을 우선한다.

- 얼굴 방향 기반 포인터 이동
- 시선 방향 보조 입력
- 화면 구역 기반 이동
- 포인터 이동 속도 조절
- 포인터 떨림을 줄이기 위한 smoothing

## 4.2. 기본 터치

사용자가 음성으로 "터치" 또는 "클릭"이라고 말하면 현재 포인터 위치에서 터치 이벤트를 수행한다.

지원 명령 예시:

| 음성 명령 | 동작 |
| --- | --- |
| 터치 | 현재 위치 탭 |
| 클릭 | 현재 위치 탭 |
| 취소 | 현재 동작 취소(뒤로가기) |

## 4.3. 드래그

드래그는 손 없는 조작에서 가장 중요한 기능 중 하나이다. 드래그는 상/하 화면 이동을 의미한다. MVP에서는 단순히 길게 말하는 방식으로 구현한다.

예시:

1. 포인터를 드래그 시작 위치로 이동한다.
2. 사용자가 "위 드래그/아래 드래그"이라고 말한다.
3. “드래그으으으”라고 길게 말하는 동안 드래그된다.
4. 사용자가 말을 그만하면 드래그가 종료된다.

지원 명령 예시:

| 음성 명령 | 동작 |
| --- | --- |
| 위 드래그으으으 | 드래그 시작 |
| 아래 드래그으으으 | 드래그 시작 |

## 4.4. 스크롤

스크롤은 웹페이지, 문서, 레시피, 영상 목록 등에서 자주 쓰이는 기능이다. 스크롤은 좌/우 화면 이동을 의미한다.

지원 명령 예시:

| 음성 명령 | 동작 |
| --- | --- |
| 왼쪽 스크로오오올 | 스크롤 시작 |
| 오른쪽 스크로오오올 | 스크롤 시작 |

## 4.5. 일시 정지 및 재개

의도하지 않은 조작을 막기 위해 컨트롤러를 언제든 멈출 수 있어야 한다.

| 음성 명령 | 동작 |
| --- | --- |
| 멈춰 | 포인터 이동 및 입력 일시 정지 |
| 다시 시작 | 컨트롤러 재개 |
| 잠금 | 입력 잠금 |
| 해제 | 입력 잠금 해제 |

## 4.6. 캘리브레이션

사용자마다 얼굴 위치, 카메라 각도, 화면 크기, 사용 자세가 다르기 때문에 초기 보정 과정이 필요하다.

캘리브레이션 흐름:

1. 기기를 거치하고 얼굴이 화면 중앙에 오도록 안내한다.
2. 화면 중앙을 바라본다.
3. 상하좌우 또는 9개 기준점을 차례대로 바라본다.
4. 앱이 사용자의 얼굴 및 눈 움직임 범위를 저장한다.
5. 테스트 포인터를 움직이며 민감도를 조절한다.

# 5. 사용 시나리오

시나리오 1: 요리 중 레시피 조작

사용자는 요리 중 손이 젖거나 음식 재료가 묻어 스마트폰을 만지기 어렵다. 스마트폰을 거치대에 올려두고 핸즈프리 컨트롤러를 실행한다.

1. 사용자가 레시피 화면을 바라본다.
2. 다음 단계가 필요하면 "아래로"라고 말한다.
3. 특정 버튼을 누르고 싶으면 해당 버튼 쪽으로 포인터를 이동한다.
4. "터치"라고 말해 버튼을 누른다.

시나리오 2: 태블릿 영상 앱 조작

사용자는 침대나 책상 위에 태블릿을 세워두고 영상을 본다.

1. 영상 화면 중앙을 바라본다.
2. "터치"라고 말해 재생/일시정지를 수행한다.
3. 목록을 보고 싶으면 "아래로"라고 말한다.
4. 원하는 영상을 바라본 뒤 "터치"라고 말한다.

시나리오 3: 손 사용이 어려운 사용자의 기본 조작

사용자는 손 사용이 제한적이지만 스마트폰의 기본 앱을 조작하고 싶다.

1. 앱을 실행하고 캘리브레이션을 진행한다.
2. 포인터를 원하는 앱 아이콘이나 버튼 위로 이동한다.
3. "터치"라고 말해 선택한다.
4. 목록이나 문서는 "위로", "아래로" 명령으로 이동한다.
5. 조작이 필요 없을 때는 "멈춰"라고 말해 오작동을 막는다.

시나리오 4: 발표 또는 수업 중 태블릿 조작

사용자는 발표 중 태블릿을 직접 만지지 않고 슬라이드나 자료를 넘기고 싶다.

1. 태블릿을 발표자 앞에 둔다.
2. "다음", "이전" 같은 명령으로 페이지를 넘긴다.
3. 특정 버튼이 필요하면 포인터를 이동한 뒤 "터치"라고 말한다.
