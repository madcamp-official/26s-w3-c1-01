# 핸즈프리 컨트롤러 — 설치 및 사용 가이드

**얼굴로 포인터를 옮기고, 목소리로 터치하는 안드로이드 앱입니다.**

요리 중이거나, 태블릿을 거치해 두고 보거나, 손을 쓰기 어려운 상황에서
화면을 만지지 않고 조작하기 위해 만들었습니다.

---

## 먼저 알아두실 것

이 앱은 **접근성 서비스 권한**을 요구합니다. 다른 앱 위에 터치를 대신 발생시키려면
안드로이드에서 이 권한 외에 다른 방법이 없기 때문입니다.

**접근성 권한은 강력한 권한입니다.** 낯선 앱에 이 권한을 주는 것을 망설이는 것이
정상이고, 저희는 그 판단이 옳다고 생각합니다. 그래서 이렇게 하고 있습니다:

- **소스 코드 전체가 공개되어 있습니다** — 직접 확인하실 수 있습니다
- 화면 내용을 읽지 않습니다. 접근성 이벤트 구독을 아예 꺼 두었습니다
  (`accessibility_service_config.xml`에서 확인 가능)
- 카메라 영상, 얼굴 이미지, 음성 녹음은 **저장하지도 전송하지도 않습니다**
- 진단 데이터 전송은 **기본이 꺼져 있고**, 켜야만 보냅니다

그래도 불안하시면 설치하지 않으셔도 됩니다. **써 보고 지우셔도 됩니다** —
앱을 삭제하면 저장된 데이터(캘리브레이션 수치, 식별자)도 함께 사라집니다.

---

## 준비물

| | |
| --- | --- |
| 안드로이드 | **8.0 이상** (Oreo, 2017년 이후 대부분의 기기) |
| 카메라 | **전면 카메라 필수** |
| 네트워크 | **음성 인식에 인터넷이 필요합니다** (아래 설명) |
| 저장 공간 | 약 150MB |
| 설치 파일 | 약 55MB |

> **음성 인식에 인터넷이 필요합니다.** 원래는 기기 안에서만 처리하려 했지만,
> 한국어 오프라인 음성 모델이 없는 기기에서 인식이 전혀 되지 않아 온라인 방식으로
> 되돌렸습니다. 즉 **음성 명령이 구글의 음성 인식을 거칩니다.** 이 부분은 안드로이드
> 기본 음성 인식 기능을 그대로 쓰는 것이며, 저희 서버로 가는 것은 아닙니다.

**아이폰에서는 동작하지 않습니다.** iOS는 다른 앱 위에 터치를 발생시키는 것을
허용하지 않습니다. 안드로이드 전용입니다.

## 1. 설치하기

### 1-1. APK 내려받기

`app-release.apk` 파일을 기기에 내려받습니다. (배포 링크는 저장소 최상단 참고)

### 1-2. "알 수 없는 출처" 허용

Play 스토어가 아닌 곳에서 받은 앱이라 한 번 허용해 주셔야 합니다.

1. 내려받은 APK 파일을 누릅니다
2. **"이 출처의 앱 설치 차단됨"** 같은 안내가 뜹니다
3. **설정** 을 누릅니다
4. **"이 출처 허용"** 을 켭니다
5. 뒤로 가서 다시 설치를 누릅니다

> 기기에 따라 문구가 다릅니다: 삼성은 "알 수 없는 앱 설치",
> 일부 기기는 "출처를 알 수 없는 앱". 브라우저나 파일 관리자 앱별로 따로 물어봅니다.

설치가 끝나면 **"핸즈프리 컨트롤러"** 아이콘이 생깁니다.

---

## 2. 권한 설정 (처음 한 번만)

앱을 열면 **1 → 2 → 3 순서대로** 버튼이 있습니다. **순서대로 누르셔야 합니다.**

### 1️⃣ 접근성 서비스 켜기

버튼을 누르면 안드로이드 설정 화면으로 이동합니다.

1. 목록에서 **"핸즈프리 컨트롤러"** 를 찾습니다
   (설치된 앱 / 다운로드한 앱 / 설치된 서비스 등의 항목 아래에 있습니다)
2. 눌러서 **켬** 으로 바꿉니다
3. 경고 문구가 뜨면 **허용** 을 누릅니다
4. **뒤로 가기**로 앱에 돌아옵니다

> **이 단계를 건너뛰면 음성 명령이 하나도 실행되지 않습니다.** 포인터는 움직이는데
> "터치"라고 말해도 아무 일도 일어나지 않으면 이게 원인입니다.

### 2️⃣ 오버레이 권한 / 시작

버튼을 누르면 **"다른 앱 위에 표시"** 설정으로 이동합니다. 켜고 돌아옵니다.

이게 있어야 다른 앱을 쓰는 동안에도 포인터가 화면에 보입니다.

### 3️⃣ 시작

**카메라**와 **마이크** 권한을 묻습니다. 둘 다 **허용**을 누릅니다.

---

## 3. 캘리브레이션 (처음 한 번만)

포인터를 정확히 움직이려면 **고개를 어디까지 돌리는지**를 앱이 알아야 합니다.

시작을 누르면 화면에 점이 하나씩 나타납니다. **약 22초** 걸립니다.

**하는 방법:**

1. 기기를 평소 쓰는 자세로 두세요 (손에 들거나, 거치하거나 — 실제 사용할 자세로)
2. 화면에 나타나는 점을 **고개를 돌려서** 바라봅니다
3. 점이 사라지고 다음 점이 나올 때까지 **그 자세를 유지**합니다
4. 9개 점을 모두 마치면 자동으로 시작됩니다

**잘 되게 하는 요령:**

- **눈만 굴리지 말고 고개를 돌리세요.** 이 앱은 고개 방향이 주 입력입니다
- 화면 가장자리 점에서는 **실제로 그 방향까지 충분히 돌리세요.** 여기서 아낀 만큼
  나중에 화면 끝까지 포인터가 안 갑니다
- 밝은 곳에서 하세요. 어두우면 얼굴 인식이 흔들립니다

**다시 하고 싶으면** 앱의 **"캘리브레이션 다시 하기"** 버튼을 누르세요.
자세나 거치 위치를 바꿨다면 다시 하는 게 좋습니다.

> 한 번 하면 저장됩니다. 앱을 껐다 켜도 다시 하지 않습니다.

---

## 4. 사용하기

이제 앱을 나가도 됩니다. **포인터가 화면 위에 계속 떠 있습니다.**

고개를 움직여 포인터를 옮기고, 아래 명령을 **말해서** 실행합니다.

### 음성 명령

| 하고 싶은 것 | 말할 것 |
| --- | --- |
| **누르기** | "터치" / "클릭" |
| 뒤로 가기 | "취소" |
| **아래로 스크롤** | "아래로" / "내려" |
| **위로 스크롤** | "위로" / "올려" |
| 다음 화면 (좌우) | "다음" / "오른쪽" |
| 이전 화면 (좌우) | "이전" / "왼쪽" |
| **화면 잠그기** | "잠금" / "끝" / "멈춰" |
| 잠금 풀기 | "해제" |
| 앱 종료 | "종료" |

**설계상 중요한 점: 쳐다보는 것만으로는 아무것도 실행되지 않습니다.**
화면을 그냥 보는 것과 조작하려는 의도를 구분하기 위해, 실행은 전부 음성으로만 됩니다.

### 잠그고 싶을 때

- **"잠금" / "끝" / "멈춰"** — 포인터와 상태 표시가 사라집니다.
- **"해제"** — 다시 사용할 수 있습니다.

> **음성이 안 먹혀도 갇히지 않습니다.** 화면에 **수동 잠금 해제 버튼**이 항상
> 표시되며, 손으로 눌러서 풀 수 있습니다.

### 완전히 끄기

세 가지 방법 중 아무거나:

- 알림창을 내려서 **"정지"** 를 누릅니다
- 앱을 열어 **"정지 (카메라, 마이크 끄기)"** 를 누릅니다
- 앱을 완전히 종료합니다

끄면 카메라와 마이크가 함께 꺼집니다.

## 5. 잘 안 될 때

### 포인터가 안 보여요

- 2번 **오버레이 권한**이 켜져 있는지 확인하세요
- 알림창에 앱 알림이 떠 있는지 보세요. 없으면 서비스가 꺼진 것입니다

### 포인터는 움직이는데 "터치"가 안 먹혀요

**대부분 1번 접근성 서비스가 꺼진 경우입니다.** 설정에서 다시 확인해 주세요.
기기를 재시작하거나 앱을 업데이트하면 **꺼지는 경우가 있습니다.**

### 음성 인식이 안 돼요

- **인터넷 연결**을 확인하세요 (음성 인식에 필요합니다)
- 명령어를 **또박또박, 짧게** 말해 보세요
- 주변이 시끄러우면 인식률이 떨어집니다

### 포인터가 반대로 움직여요 / 화면 끝까지 안 가요

**캘리브레이션을 다시 해 보세요.** 앉은 자세나 기기 위치가 바뀌면 흔히 생깁니다.
다시 할 때 **가장자리 점에서 고개를 충분히 돌리는 것**이 핵심입니다.

### 포인터가 떨려요 / 너무 느려요

밝은 곳에서 사용해 보세요. 조명이 어두우면 얼굴 인식이 불안정해집니다.

### 기기가 뜨거워지고 느려져요

카메라와 마이크를 계속 켜 두기 때문에 발열이 있습니다. **오래 쓰실 때는 중간에
"정지"로 꺼 주세요.** 저희도 개선해야 할 부분으로 보고 있습니다.

## 6. 알려진 한계 (버그가 아니라 알고 있는 제약입니다)

### 눈만으로는 조작할 수 없습니다

**고개 방향이 주 입력이고, 눈동자는 미세 조정 역할만 합니다.** 일반 전면 카메라로
시선을 정밀하게 추적하는 것은 물리적으로 어렵습니다. "눈으로만 화면을 탐색"하는
것을 기대하셨다면 그건 이 앱이 하는 일이 아닙니다.

### 홈 화면에서 좌우 페이지 넘김이 안 됩니다

**"다음"/"이전"이 갤러리나 일반 앱에서는 동작하는데, 삼성 런처 홈 화면에서는
페이지가 넘어가지 않습니다.** 홈 화면에서 앱을 여는 터치는 정상 동작합니다.
런처가 페이지 넘김에 자체 제스처 처리를 써서 저희가 보내는 신호를 받지 않는
것으로 파악했고, 앱 쪽에서 고칠 방법을 찾지 못했습니다.

### 주변 사람의 말에 반응할 수 있습니다

누가 말했는지 구분하지 않습니다. 옆에서 "아래로"라고 말하면 스크롤될 수 있습니다.

### 음성 인식에 인터넷이 필요합니다

위 "준비물" 항목의 설명을 참고해 주세요.

## 7. 개인정보

### 저장하지도, 전송하지도 않는 것

- **카메라 영상과 얼굴 이미지** — 화면 좌표를 계산하는 즉시 버립니다
- **음성 녹음** — 저장하지 않습니다
- **화면 내용** — 읽지 않습니다 (접근성 이벤트 구독을 꺼 두었습니다)
- 이름, 전화번호, 연락처 등 개인정보 — 요구하지 않습니다

### 기기에만 저장되는 것

- 캘리브레이션 수치 (고개 각도 범위 같은 숫자입니다)
- 설치 식별자 (무작위 값)

**앱을 삭제하면 함께 사라집니다.**

### 동의하신 경우에만 전송되는 것

앱 안에서 **"진단 데이터 전송에 동의합니다"** 를 켜신 경우에만 보냅니다.
**기본값은 꺼짐입니다.**

- 앱 실행 / 캘리브레이션 성공·실패 / 명령 성공·실패 / 성능 수치 / 오류 내용
- 기기 모델명, 안드로이드 버전, 앱 버전
- 직접 작성해 주신 피드백
- 사용자 수를 세기 위한 **무작위 식별자**(설치마다 하나, 앱을 지우면 사라집니다)

**얼굴이나 음성은 동의하셔도 보내지 않습니다.** 보내는 코드 자체가 없습니다.

---

## 8. 피드백 부탁드립니다

**이 앱은 만들다 만 것에 가깝습니다.** 저희가 가진 기기 몇 대에서만 확인했고,
다른 기기에서 어떻게 동작하는지 모릅니다. 안 되는 게 있으면 그게 저희에게
가장 필요한 정보입니다.

앱 화면 아래쪽 **"불편했던 점을 적어 주세요"** 칸에 적어 주시면 됩니다
(진단 데이터 전송에 동의하신 경우 전달됩니다).

이런 내용이 특히 도움이 됩니다:

- **기기 모델명** (예: 갤럭시 S21)
- 어떤 앱에서 무엇을 하려다 안 됐는지
- 인식이 안 된 명령어가 있다면 무엇인지
- 캘리브레이션이 잘 안 됐다면 어느 단계에서인지

깃허브 이슈로 남겨 주셔도 좋습니다.
