# 기능 명세서 (Functional Specification) v1.0

## 핸즈프리 스마트폰 컨트롤러 · 대상 범위: MVP(최초 실험 버전)

> 기획안 원문은 [PLAN.md](PLAN.md). **수정 시 팀 합의 필요.**

## 1. 문서 개요

### 1.2 범위

**포함(MVP)**: 얼굴/시선 기반 포인터 이동 · 음성 명령 기반 터치/드래그/스크롤 ·
일시정지/재개, 입력 잠금 · 초기 캘리브레이션 · 음성 인식 처리, 상태 관리, 예외 처리

**제외(Phase 2 이후)**: 특정 앱 전용 최적화 · 정밀 아이트래킹(각막 반사 등) ·
다중 사용자 프로필 클라우드 동기화 · 제스처(표정, 눈 깜빡임) 기반 명령

### 1.3 용어 정의

| 용어 | 정의 |
| --- | --- |
| 포인터 | 얼굴/시선 추정 결과를 화면 좌표에 매핑해 표시하는 가상 커서 |
| 트리거 명령 | 포인터 위치에서 실제 동작을 실행시키는 음성 명령 |
| 캘리브레이션 | 얼굴/눈 움직임 범위와 화면 좌표를 매핑하는 초기 보정 |
| ACTIVE | 포인터 이동 및 음성 명령이 모두 처리되는 상태 |
| PAUSED | 포인터 이동은 멈추고, 재개 명령만 인식하는 상태 |
| LOCKED | 모든 입력이 무시되는 상태 |
| Dwell(체류) | 포인터가 일정 시간 머무는 것. **트리거 보조 용도로만, 자동 실행에는 미사용** |

## 2. 시스템 개요

### 2.1 주요 모듈

| 모듈 | 역할 |
| --- | --- |
| Face Tracker | 전면 카메라 입력에서 얼굴 방향, 눈 위치, 머리 움직임 추정 |
| Pointer Mapper | 추정값을 화면 좌표로 변환, smoothing 및 속도 조절 |
| Voice Recognizer | 사용자 발화를 텍스트/명령으로 변환 |
| Command Interpreter | 인식 텍스트를 사전 정의 명령으로 매핑, 동의어 처리 |
| State Manager | ACTIVE/PAUSED/LOCKED/CALIBRATING/DRAGGING 상태 전이 관리 |
| Input Injector | OS 접근성 서비스를 통해 실제 탭/드래그/스크롤 이벤트 발생 |
| Calibration Manager | 초기 보정 플로우 및 사용자 프로파일 저장 |
| Overlay UI | 포인터, 상태 인디케이터, 설정 화면 표시 |

### 2.2 전체 동작 흐름

```
[카메라 입력] → [얼굴/시선 추정] → [포인터 좌표 계산] → [오버레이에 포인터 표시]
                                                              ↑
[마이크 입력] → [음성 인식] → [명령 해석] → [상태 확인] → [입력 실행] → [OS 이벤트]
```

포인터 이동과 음성 명령 처리는 **병렬**로 동작하며, **명령 실행 시점의 포인터 좌표**를
기준으로 동작이 수행된다.

### 2.3 상태 정의

| 상태 | 진입 조건 | 진출 조건 |
| --- | --- | --- |
| CALIBRATING | 앱 최초 실행, 재보정 요청 | 보정 완료 |
| ACTIVE | 보정 완료, "다시 시작", "해제" | "멈춰", "잠금", 드래그 시작 |
| PAUSED | "멈춰" | "다시 시작" |
| LOCKED | "잠금" | "해제" |
| DRAGGING | "잡아"/"드래그 시작" | "놓아", "드래그 취소" |

## 3. 기능 명세

### FR-001 포인터 이동

**입력**: 전면 카메라 프레임 스트림(권장 15fps 이상), 캘리브레이션 프로파일

**처리**: 얼굴 랜드마크 검출 → 얼굴 방향(yaw/pitch)·눈 위치 추정 → 프로파일 기준
정규화 → 화면 좌표 매핑 → Smoothing 필터(이동 평균/저역 통과) → 속도 설정값 스케일링

**세부 파라미터**

| 파라미터 | 설명 | 기본값 | 조정 범위 |
| --- | --- | --- | --- |
| 이동 속도(Sensitivity) | 얼굴 각도 변화 대비 포인터 이동량 | 중간 | 낮음/중간/높음 |
| Smoothing 강도 | 떨림 저감 필터 강도 | 중간 | 낮음/중간/높음 |
| 화면 구역 이동 | 화면을 N×N 구역으로 분할 | 3×3 | 2×2 ~ 4×4 |
| 시선 보조 가중치 | 얼굴 방향 대비 시선 반영 비율 | 낮음 | 0~100% |

**예외**

| 상황 | 처리 |
| --- | --- |
| 얼굴 미검출 | 포인터를 **마지막 위치에 고정**, "얼굴 인식 안됨" 안내 |
| 저조도 | 밝기 부족 안내, 화면 밝기 자동 보정 제안 |
| 캘리브레이션 범위 초과 | 포인터를 화면 경계값으로 **클램핑** |
| 카메라 권한 거부 | 사용 불가 안내 + 권한 설정 화면 유도 |

### FR-002 기본 터치

**처리**: 명령 인식 → **현재 상태가 ACTIVE인지 확인**(PAUSED/LOCKED이면 무시) →
포인터 좌표에서 OS 탭 이벤트 생성 → 시각/청각 피드백

| 음성 명령 | 동작 | 비고 |
| --- | --- | --- |
| 터치 | 현재 위치 탭 | 기본 명령 |
| 클릭 | 현재 위치 탭 | 동의어 |
| 취소 | 뒤로가기 | OS 뒤로가기 이벤트 매핑 |

**예외**: PAUSED/LOCKED에서는 실행하지 않고 상태 안내만 표시.
포인터가 클램핑된 상태면 가장 가까운 유효 좌표에서 탭.

### FR-003 드래그

> ⚠️ 기획안 4.3(상/하 이동 + 지속 발화)과 충돌. 아래는 **명세서 안(이산 명령)**.

**처리**
1. "잡아"/"드래그 시작" → DRAGGING 전이, 현재 좌표를 시작점으로 기록
2. DRAGGING 중 포인터 이동에 따라 터치 다운 유지 + 좌표 갱신
3. "놓아" → 현재 좌표에서 터치 업, ACTIVE 복귀
4. "드래그 취소" → 시작점으로 포인터 복귀, 터치 업 없이 취소, ACTIVE 복귀

**예외**

| 상황 | 처리 |
| --- | --- |
| DRAGGING 중 "터치"/"클릭" | 무시 또는 "드래그 중입니다" 안내 |
| DRAGGING 중 얼굴 미검출 | 마지막 좌표 고정, **드래그 상태 유지(자동 취소 안 함)** |
| DRAGGING 30초 이상 지속 | 자동 "드래그 취소" 처리 후 안내 |

### FR-004 스크롤

**처리**: 방향(위/아래)과 강도(기본/조금/크게)를 함께 파싱 → 강도별 스크롤 거리 결정
→ 포인터 좌표 기준 OS 스와이프 이벤트 생성

| 음성 명령 | 스크롤 거리(화면 높이 대비) |
| --- | --- |
| 아래로 / 위로 | 약 50% |
| 조금 아래로 / 조금 위로 | 약 20% |
| 크게 아래로 / 크게 위로 | 약 80% |

**예외**: 스크롤 불가 화면은 별도 처리 없이 OS/앱단에 위임.
연속 발화("아래로 아래로")는 개별 명령으로 순차 처리.

### FR-005 일시정지 및 재개

| 구분 | PAUSED | LOCKED |
| --- | --- | --- |
| 포인터 표시 | 마지막 위치 고정 | 숨김 또는 반투명 |
| 인식되는 명령 | "다시 시작"만 | "해제"만 |
| 용도 | 일시적 정지 | 장시간 미사용, 오조작 완전 차단 |

**예외**
- LOCKED에서 다른 명령 발화 시 무시 + "잠금 해제가 필요합니다" 안내
- **음성 인식 오류로 "해제"가 반복 실패할 경우를 대비해, 화면 내 수동 잠금 해제
  버튼(물리 터치)을 항상 노출** ← 안전장치, 제거 불가

### FR-006 캘리브레이션

| 단계 | 내용 |
| --- | --- |
| 1 | 기기 거치 및 얼굴 정렬 안내 |
| 2 | 화면 중앙 응시 |
| 3 | 상/하/좌/우 및 대각선 총 **9개 기준점** 순차 응시 |
| 4 | 얼굴/눈 움직임 범위 저장 (진행률 표시) |
| 5 | 테스트 포인터로 민감도 조정 |

**프로파일 스키마(안)**: `profile_id` · `reference_points[9]`(기준점별 원시값) ·
`face_range_yaw_min/max` · `face_range_pitch_min/max` · `sensitivity_level` ·
`smoothing_level` · `created_at`/`updated_at` — **기기 로컬 저장**

**예외**

| 상황 | 처리 |
| --- | --- |
| 기준점에서 얼굴 미검출 5초 이상 | 해당 단계 재시도, 3회 실패 시 처음부터 재시작 |
| 중도 이탈 | 임시 저장 없이 폐기, 다음 실행 시 재시작 |
| 미완료 상태로 앱 실행 | 자동으로 캘리브레이션 진입, **ACTIVE 진입 차단** |
| 조명/각도 문제로 반복 실패 | 조명 개선·거치 각도 가이드 문구 제공 |

**재보정**: 설정에서 선택 시 기존 프로파일을 덮어씀. 재보정 중에는 기존 프로파일로
계속 조작 가능(완료 후 교체).

### FR-007 음성 명령 인식 및 해석

**모든 실제 조작의 트리거는 음성으로만 처리하며, 시선/얼굴 위치만으로 자동 실행되지 않는다.**

**처리**: 마이크 입력 → 실시간 인식 → 명령어 사전 매칭(동의어 포함) →
State Manager 유효성 검증 → 실행

**원칙**: 각 명령은 최소 1개 이상의 동의어 지원. 사전은 확장 가능한 구조로 설계하되
MVP에서는 기본 사전만 제공(커스터마이징은 Phase 2).

**예외**

| 상황 | 처리 |
| --- | --- |
| 사전에 없는 텍스트 | 동작 없음 + "인식된 명령 없음" 짧게 표시 |
| 소음 환경 | 신뢰도 임계값 이하면 명령 무시, 재발화 유도 |
| 타인 발화 오작동 | MVP는 **임계값 기반 필터링만**. 화자 인식은 Phase 2 |
| 마이크 권한 거부 | 사용 불가 안내 + 권한 설정 화면 유도 |

### FR-008 상태 관리 및 오버레이 표시

| 상태 | 포인터 | 인디케이터 |
| --- | --- | --- |
| ACTIVE | 실시간 이동, 기본 색상 | 초록 |
| PAUSED | 마지막 위치 고정, 반투명 | 노랑 + "일시정지" |
| LOCKED | 숨김/매우 반투명 | 회색 + "잠김" |
| DRAGGING | 이동 중, 강조 색상 | 파랑 + "드래그 중" |
| CALIBRATING | 기준점 전용 UI | 진행률 |

**예외**: 상태 전이 충돌(드래그 중 "멈춰") 시 **안전 우선 — "멈춰"를 우선 처리하고
드래그는 자동 취소 후 PAUSED로 전이**.

## 4. 전체 음성 명령어 통합 목록

| 분류 | 음성 명령 | 동작 | 유효 상태 |
| --- | --- | --- | --- |
| 터치 | 터치 / 클릭 | 현재 위치 탭 | ACTIVE |
| 터치 | 취소 | 뒤로가기 | ACTIVE |
| 드래그 | 잡아 / 드래그 시작 | 드래그 시작 | ACTIVE |
| 드래그 | 놓아 | 드래그 종료 | DRAGGING |
| 드래그 | 드래그 취소 | 드래그 취소 | DRAGGING |
| 스크롤 | 아래로 / 위로 | 기본 스크롤 | ACTIVE |
| 스크롤 | 조금 아래로 / 조금 위로 | 짧은 스크롤 | ACTIVE |
| 스크롤 | 크게 아래로 / 크게 위로 | 긴 스크롤 | ACTIVE |
| 제어 | 멈춰 | 일시정지 | ACTIVE, DRAGGING |
| 제어 | 다시 시작 | 재개 | PAUSED |

> 좌/우 스와이프 이벤트로 매핑되는 명령으로 정의(FR-004에 준하는 처리)할 것을 제안.

## 5. 비기능 요구사항 (NFR)

| 구분 | 요구사항 |
| --- | --- |
| 지연시간 | 얼굴 인식~포인터 반영까지 **200ms 이내** |
| 프레임 | 얼굴 추정 처리 **최소 15fps** |
| 배터리/발열 | 카메라+음성 동시 사용 시 발열/소모 모니터링, 절전 옵션 제공 |
| 개인정보보호 | **온디바이스 처리 원칙. 원본 영상/음성 저장 안 함**(캘리브레이션 수치만 로컬) |
| 접근성 | 손 사용 제한 사용자가 최우선 대상. 명령어는 발음하기 쉬운 짧은 단어 위주 |
| 안전장치 | **"멈춰"/"잠금"은 항상 최우선 처리**(다른 명령과 충돌해도 즉시 반영) |
| 오프라인 | 네트워크 없이 동작 가능(온디바이스 음성/시선 인식) |
