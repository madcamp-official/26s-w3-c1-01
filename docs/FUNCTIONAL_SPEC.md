# 기능 명세서 (Functional Specification) v1.1

## 핸즈프리 스마트폰 컨트롤러 · MVP 현재 범위

## 1. 범위

**포함(MVP)**: 얼굴/시선 기반 포인터 이동, 음성 명령 기반 터치/뒤로가기/스크롤/좌우 이동, 입력 잠금, 초기 캘리브레이션, 상태 관리, 예외 처리.

**제외**: 드래그, 일시정지/재개, 스크롤 강도 명령, 특정 앱 전용 최적화, 정밀 아이트래킹, 사용자 정의 명령어.

## 2. 상태 정의

| 상태 | 의미 | 진입 | 진출 |
| --- | --- | --- | --- |
| CALIBRATING | 보정 중. 모든 음성 명령 폐기 | 최초 실행, 재보정 | 보정 완료 |
| ACTIVE | 포인터 표시 및 유효 명령 실행 | 보정 완료, 해제 | 잠금 |
| LOCKED | 포인터/HUD 숨김. 잠금 해제만 허용 | 잠금 | 해제 |

## 3. 명령 정책

`CommandDictionary`는 B 파트의 문자열 사전이고, D 파트는 `CommandId` enum만 본다.

| CommandId | 한국어 명령 | 동작 | 유효 상태 |
| --- | --- | --- | --- |
| TOUCH | 탭, 터치, 클릭 | 현재 포인터 위치 탭 | ACTIVE |
| BACK | 취소, 뒤로 | OS 뒤로가기 | ACTIVE |
| SCROLL_DOWN | 아래로, 내려, 아래 | 아래 방향 스크롤 | ACTIVE |
| SCROLL_UP | 위로, 올려, 위 | 위 방향 스크롤 | ACTIVE |
| NEXT | 다음, 오른쪽 | 오른쪽/다음 페이지 이동 | ACTIVE |
| PREV | 이전, 왼쪽 | 왼쪽/이전 페이지 이동 | ACTIVE |
| LOCK | 잠금, 끝, 멈춰 | LOCKED 전이 | ACTIVE |
| UNLOCK | 해제 | ACTIVE 전이 | LOCKED |
| EXIT | 종료 | 앱/서비스 종료. D 상태머신을 거치지 않음 | 모든 상태 |

폐기된 명령어: `잡아`, `드래그 시작`, `놓아`, `놔`, `드래그 취소`, `조금 아래로`, `조금 위로`, `크게 아래로`, `크게 위로`, `다시 시작`.

## 4. 주요 기능

### FR-001 포인터 이동

전면 카메라 프레임에서 얼굴 방향과 눈 위치를 추정하고, 캘리브레이션 프로파일 기준으로 화면 좌표에 매핑한다. 얼굴이 미검출되면 마지막 유효 위치를 유지하고 오버레이에서 "얼굴 인식 안 됨" 상태를 표시한다.

### FR-002 기본 터치/뒤로가기

ACTIVE 상태에서 `TOUCH`는 현재 포인터 좌표에 탭을 주입하고, `BACK`은 접근성 서비스의 뒤로가기 액션을 실행한다.

### FR-003 스크롤/좌우 이동

ACTIVE 상태에서 `SCROLL_DOWN`, `SCROLL_UP`, `NEXT`, `PREV`를 접근성 제스처로 실행한다. 스크롤 강도 명령은 MVP에서 제거했다.

### FR-004 잠금/해제

`LOCK`은 화면 조작을 잠그고 포인터/HUD를 숨긴다. LOCKED 상태에서는 `UNLOCK`만 유효하다. 음성 해제가 실패할 경우를 대비해 오버레이의 수동 잠금 해제 버튼은 유지한다.

### FR-005 캘리브레이션

9개 기준점에서 얼굴 방향 샘플을 수집해 로컬 프로파일로 저장한다. 저장된 프로파일이 있으면 다음 실행 시 보정을 건너뛰고 ACTIVE로 진입한다.

## 5. 개인정보 및 진단 데이터

카메라 영상, 얼굴 이미지, 음성 녹음, 화면 내용은 저장하지 않는다. 사용자가 진단 데이터 전송에 동의한 경우에만 앱 실행, 보정 결과, 명령 실행 결과, 성능 요약, 오류, 사용자 입력 피드백을 전송한다.
